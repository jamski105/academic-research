# v6.2 · F16 — OA-Site-Subagenten: tib-fetcher, oapen-fetcher, doabooks-fetcher, kvk-fetcher

**Audit-Bezug:** [`docs/AUDIT-v6-roadmap.md`](../blob/main/docs/AUDIT-v6-roadmap.md) §15a F16.1

## Scope
- [ ] `agents/tib-fetcher.md` (Sonnet, browser-use only) — tib.eu Portal, OA-Filter, Volltext-Download. Skelett im Spec §15a F16.1.
- [ ] `agents/oapen-fetcher.md` (Sonnet) — oapen.org, reine OA-Buch-Plattform, kein Auth
- [ ] `agents/doabooks-fetcher.md` (Sonnet) — directory.doabooks.org, REST-Discovery via Browser
- [ ] `agents/kvk-fetcher.md` (Sonnet) — kvk.bibliothek.kit.edu Meta-Suche über 80+ Kataloge

## Pro Subagent enthalten
- `tools: [Bash(browser-use:*), Bash(browser-use *), Read, Write]`
- maxTurns: 12-15
- System-Prompt mit Standard-Flow + Verbote (kein curl/API)
- Output: `{status: success|metadata_only|captcha|no_match, pdf_path, url}`

## Akzeptanzkriterien
- [ ] Pro Subagent: 3 Test-ISBNs/DOIs → erwartetes Output-Schema
- [ ] Browser-use download-Action speichert PDFs in `<session>/pdfs/`
- [ ] Validation: PDF-Magic-Bytes + Größe > 10 KB
