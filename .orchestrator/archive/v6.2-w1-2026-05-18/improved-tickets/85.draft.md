## Context

`/academic-research:fetch` ist der User-Entrypoint des Universal Book Fetcher-Systems (F16, v6.2). Der Command nimmt einen beliebigen Identifier (ISBN, DOI, Titel, URL) entgegen, parst ihn, spawnt den Master-Subagenten `book-fetcher` und integriert das Ergebnis in Vault/`literature_state.md`. Das System setzt die in v6.1 fertiggestellte Buch-Pipeline voraus (`book-handler`, `chunk_pdf`, `page_offset`, OCR-Detection, CSL-Schema).

Bezug: [`docs/AUDIT-v6-roadmap.md`](../blob/main/docs/AUDIT-v6-roadmap.md) §15a F16.3. Master-Agent `book-fetcher` wird in Issue #80 spezifiziert.

Die YAML-Frontmatter des Commands folgt der Repo-Konvention für Slash-Commands: `allowed-tools: Read, Write, Agent(book-fetcher)`, `argument-hint: <isbn|doi|titel|url>`.

## Acceptance Criteria

- [ ] `commands/fetch.md` existiert mit gültigem YAML-Frontmatter (`description`, `allowed-tools: Read, Write, Agent(book-fetcher)`, `argument-hint: <isbn|doi|titel|url>`)
- [ ] Input-Parsing erkennt korrekt: ISBN-10/ISBN-13-Pattern, DOI-Pattern (`10.`-Prefix), HTTP/HTTPS-URL, sowie Freitext als Titel-Fallback
- [ ] Der Command spawnt `book-fetcher` mit dem normalisierten Input und wartet auf dessen Ergebnis
- [ ] Bei Status `success`: PDF-Pfad und Metadaten werden in `literature_state.md` und Vault aufgenommen
- [ ] Bei Status `pickup_required`: Ein Eintrag für die pickup-list.xlsx-Pipeline wird vorbereitet (Aufnahme in die F5-Fernleihe-Pipeline)
- [ ] Bei Status `captcha`: Der Screenshot wird dem User angezeigt, der Command pausiert und wartet auf manuelle Entscheidung
- [ ] Command-Eval mit 3 Test-Inputs (je einem ISBN-, DOI- und URL-Input) läuft durch und liefert dokumentiertes Ergebnis; Eval-Dateien liegen unter `evals/` oder `specs/v6.2/`
- [ ] Help-Text / `description`-Feld im Frontmatter beschreibt Argumentformat und mögliche Ausgabe-Stati für den User

## Out of Scope

- Implementierung der Site-Subagenten (`tib-fetcher`, `springer-book-fetcher` usw.) — diese sind in F16.1 (Issue #80) spezifiziert
- Implementierung des Master-Agents `book-fetcher` — ebenfalls Issue #80
- Erstellung der pickup-list.xlsx selbst — das ist F5 (`commands/pickup.md`)
- Per-Uni-Profil-Setup und `auth-helper` — F16.5
