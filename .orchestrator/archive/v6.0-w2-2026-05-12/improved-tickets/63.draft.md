## Context

Phase 1 (#62, gemergt) hat den `academic-vault` MCP-Server mit SQLite-Schema (`papers`, `quotes`, `quote_embeddings`, `decisions`, `notes`, `papers_fts`) und den Tools `vault.search`, `vault.get_paper`, `vault.add_paper`, `vault.ensure_file`, `vault.add_quote` (nur mit `extraction_method='citations-api'` + `api_response_id`), `vault.find_quotes`, `vault.get_quote` und `vault.stats` geliefert.

Phase 2 verdrahtet die bestehenden Agents/Skills gegen diesen Vault. Heute lädt `chapter-writer` `literature_state.md` (alle Quellen-Stubs) + PDF-Auszüge manuell pro Aufruf — laut `docs/AUDIT-v6-vault.md` §4 rund 8–15 k Token Boilerplate. Mit gezielten Vault-Tool-Calls sinkt das auf ~1 700 Token (~83 % Ersparnis). Halluzinationsschutz: ein `PreToolUse`-Hook für `Write` auf `kapitel/*.md` parst zitierte Verbatim-Strings und blockt den Schreibvorgang, wenn das Zitat nicht in der `quotes`-Tabelle liegt.

## Acceptance Criteria

- [ ] `agents/quote-extractor.md` schreibt extrahierte Zitate ausschliesslich via `vault.add_quote()` mit gefülltem `api_response_id`-Feld (kein JSON-File-Schreibpfad mehr)
- [ ] `skills/citation-extraction/SKILL.md` liest Zitate ausschliesslich via `vault.find_quotes()` und `vault.get_quote()` (kein direkter PDF-Lesepfad)
- [ ] `skills/chapter-writer/SKILL.md` liest via `vault.search()` + `vault.find_quotes()` statt des vollständigen `literature_state.md`
- [ ] PDFs werden via `vault.ensure_file()` als `file_id` übergeben — kein base64 mehr im Context
- [ ] `literature_state.md` wird nur noch als read-only Snapshot-Export aus dem Vault generiert (kein Direktschreib-Pfad aus Skills/Agents)
- [ ] Token-Boilerplate in `chapter-writer` liegt unter 2 000 Token (vorher ~10 k)
- [ ] Eval bestätigt identische Zitat-Qualität bei mindestens −75 % Token-Verbrauch gegenüber dem v5.4-Baseline. Baseline wird als manueller Token-Count-Snapshot im PR-Body dokumentiert (Vor/Nach-Vergleich, keine automatisierte Eval-Suite in diesem Ticket)

## Out of Scope

- `PreToolUse`-Hook für Verbatim-Validation auf `kapitel/*.md` bleibt Scope von #64 (Vault Hook). Dieses Ticket verdrahtet nur Lese-/Schreibpfade gegen den Vault.
